01 · Como funciona

Do pedido em português à reunião na agenda

Quatro etapas, todas existentes no código: o Claude interpreta, o servidor MCP traduz, a API v2 do Calendly executa e o resultado volta verificável.

1

Você pede em linguagem natural

"Agenda um diagnóstico com o convidado na quinta às 10h", "me dá um link de 30 minutos pra semana que vem", "cancela a reunião de amanhã". O Claude entende a intenção e escolhe a tool certa deste servidor.

2

O MCP traduz o pedido em chamadas

O servidor descobre o tipo de evento (calendly_list_event_types), confere os horários livres (calendly_list_available_slots) e monta a requisição: converte o horário de Brasília pra UTC, normaliza o telefone pro formato +55 e preenche as perguntas do formulário (WhatsApp, empresa, observações).

3

A API v2 do Calendly executa

Autenticado pelo seu Personal Access Token, o agendamento vai direto pela Scheduling API (POST /invitees). Links single-use nascem de POST /scheduling_links; reuniões avulsas, de POST /one_off_event_types. A janela de disponibilidade é ajustada automaticamente aos limites da API (máx. 7 dias, nunca no passado).

4

Resultado verificável, com plano B

A reunião aparece confirmada na agenda — e calendly_get_event devolve status, link Zoom e dados do convidado. Se a API recusar o agendamento direto, o servidor devolve uma URL pré-preenchida com todos os dados: abra ou envie ao convidado e confirme em um clique. Nada se perde no caminho.

02 · As 7 tools

Uma ferramenta por operação, direto do código

Cada card abaixo é uma tool real registrada no servidor — nomes exatos, comportamento exato.

calendly_list_event_types

Lista os tipos de evento ativos da conta, com URI, slug, duração, URL pública de agendamento e as perguntas do formulário de cada um.

calendly_list_available_slots

Horários livres de um tipo de evento num intervalo de datas (máx. 7 dias por chamada), devolvidos em UTC e em horário de Brasília.

calendly_schedule

Agenda a reunião pra um convidado via Scheduling API, preenchendo nome, e-mail, WhatsApp, empresa e observações. Se a API recusar, devolve URL pré-preenchida pra confirmar em um clique.

calendly_cancel

Cancela um evento agendado pelo UUID, com o motivo registrado no Calendly.

calendly_get_event

Detalhes completos de um evento: status, horários, local (incluindo link Zoom) e convidados — com links de cancelamento e reagendamento de cada um.

calendly_create_scheduling_link

Gera um link de agendamento single-use pra um tipo de evento existente: o convidado escolhe o próprio horário e o link expira após o uso. Ideal pra enviar por WhatsApp ou e-mail.

calendly_create_one_off_link

Cria uma reunião avulsa com nome, duração (15/30/60/90 min) e intervalo de datas customizados — e devolve o link single-use já com os horários disponíveis do período.

04 · Stack

Pequeno de propósito

Um arquivo, uma dependência, uma variável de ambiente — fácil de auditar, fácil de manter.

Node.js 18+ @modelcontextprotocol/sdk transporte stdio Calendly API v2 licença MIT

Um arquivo, uma variável

Todo o servidor mora no index.js. A única configuração é a env var CALENDLY_TOKEN, com o seu Personal Access Token.

Fuso sob controle

Você fala em horário de Brasília, a API recebe UTC e a resposta volta nos dois formatos — sem reunião marcada 3 horas fora do lugar.

Sem escopo extra

O identificador do usuário é extraído do próprio token, sem chamar /users/me — um escopo a menos pra conceder.

Telefone brasileiro

Número sem código de país ganha +55 automaticamente. Sem isso, o Calendly assume EUA e recusa a validação do formulário.
